# 广东推倒胡智能算牌器

基于 **DirectX 截屏 + 图像识别 + MCTS 蒙特卡洛搜索** 的麻将决策引擎，专为**广东推倒胡**规则深度优化。

## 这是什么

一个能从腾讯麻将画面中实时抓取手牌、自动分析牌面、给出最优出牌建议的 AI 助手。

核心决策支持三种模式：

| 模式 | 速度 | 精度 | 适用场景 |
| --- | --- | --- | --- |
| 启发式评估 | 极快 (< 0.1s) | 中 | 快速参考 |
| MCTS 搜索 | 较慢 (2-8s) | 高 | 关键决策 |
| DirectX 实时 | 每轮自动 | 取决于识别 | 实战使用 |

## 功能总览

### 输入能力

| 输入方式 | 说明 |
| --- | --- |
| 手动编码 | `python main.py --hand 123m555678p1122s` |
| DirectX 实时截屏 | `python main.py --live`，自动锁定腾讯麻将窗口抓取 |
| 可见牌/碰杠信息 | `--visible` `--melds` 参数，用于局势分析 |

### 分析能力

* 牌面结构识别（顺子、对子、刻子、搭子、孤张）
* 有效进张数与改良牌计算
* 向听数估算
* 绝张 / 剩余张数追踪

### 路线评估

系统会为手牌同时评估以下路线，并计入综合评分：

| 路线 | 触发条件 |
| --- | --- |
| 碰碰胡 | 对子 + 刻子数量达标 |
| 清一色 / 混一色 | 单花色集中度 |
| 七小对 / 豪华七对 | 对子型路线（门清） |
| 十三幺 | 幺九 + 字牌收集 |
| 大三元 / 小三元 | 中发白刻子 |
| 大四喜 / 小四喜 | 东南西北风牌 |
| 混幺九 / 清幺九 / 字一色 | 幺九字牌路线 |

### 推倒胡关键规则（引擎已内建）

| 规则 | 说明 | 引擎行为 |
| --- | --- | --- |
| 不能吃牌 | 无法用上家的牌组成顺子 | 不计算"吃"分支 |
| 可以碰 / 杠 | 可碰（2张）、可明杠/暗杠/补杠 | 评估碰杠收益并计入权重 |
| **多对子 ≠ 自摸** | 手上同时有 3 对以上对子时，摸到第 4 张匹配牌**不能直接和牌**，只能碰 | 引擎正确区分"摸牌后是否构成 4面子+1雀头"，不会误判 |
| 买马 | 胡牌后从剩余牌库取 6 张，按位置映射中马翻倍 | 杠牌和碰牌受买马收益影响，权重较高 |
| 13 张 / 14 张区分 | 13 张 = 摸牌前状态，14 张 = 摸牌后切牌 | 引擎自动识别，不同状态不同输出 |

> **示例**：手牌 `1133m555678p1122s`（一万×2、三万×2、五筒×3、六七八筒顺子、一条×2、二条×2），手中有 4 组对子。
> 此时摸到一条，手牌变为 `1133m555678p11122s`，结构为 3 面子（555p 刻子 + 678p 顺子 + 111s 刻子）+ 2 对子 + 1 孤张，**不是和牌**。
> 引擎输出：**「当前未听牌，建议按排名进张」**，不会错误地显示「自摸胡牌」。

### 输出内容

```
当前手牌: 一萬 二萬 三萬 五筒 五筒 五筒 六筒 七筒 八筒 一条 一条 二条 二条
建议切牌: 三萬 (3m)
综合评分: 322.89

Top 选项:
1. 打 三萬 (3m) | 总分 322.89 | 有效进张 28
   改良牌: 一萬, 二萬, 三筒, 四筒, 五筒, 六筒, 一条, 二条
   路线评分: 碰碰胡=51, 七小对=48, 清一色趋势=12
```

所有输出均使用**中文牌名**（一萬、五筒、一条、東、中 等），括号内保留编码供核对。

## 安装

```bash
# 1. 克隆项目
git clone https://github.com/zeehomchen/majiang-ai.git
cd majiang-ai

# 2. 安装依赖
pip install opencv-python numpy pillow mss dxcam
```

## 快速开始

### 方式一：手动模式（纯编码输入）

```bash
# 基础
python main.py --hand 123m555678p1122s

# 带可见牌
python main.py --hand 123m555678p1122s --visible 3m,3m,7z

# 带牌局阶段
python main.py --hand 123m555678p1122s --phase late

# JSON 输出
python main.py --hand 123m555678p1122s --json
```

### 方式二：MCTS 搜索模式（模拟 5000 局找最优）

```bash
python main.py --hand 123m555678p1122s --mcts

# 调整模拟次数
python main.py --hand 123m555678p1122s --mcts --sims 10000

# UCB1 树搜索模式
python main.py --hand 123m555678p1122s --mcts --mcts-mode tree

# 显示搜索进度
python main.py --hand 123m555678p1122s --mcts --verbose
```

### 方式三：DirectX 实时模式

```bash
# 1. 打开腾讯麻将，进入对局界面
# 2. 运行校准（生成区域叠加图）
python main.py --calibrate

# 3. 打开 calibration_overlay.png，确认区域正确后运行实时模式
python main.py --live

# 指定窗口标题
python main.py --live --window "QQ游戏"

# 调整轮询间隔
python main.py --live --interval 2.0
```

按 `Ctrl+C` 停止。

## 项目结构

```
majiang-ai-prototype/
├── main.py                   # 入口文件
├── self_play.py              # 4-AI 差异化策略自对弈
├── personality_library.py    # 30种人格库 + 锦标赛 + 切换器
├── pyproject.toml            # 项目配置
├── README.md                 # 本文档
├── .gitignore
└── src/
    └── majiang_ai/
        ├── __init__.py       # 包定义
        ├── cli.py            # 命令行界面（三种模式）
        ├── parser.py         # 牌面编码解析（紧凑串 ↔ Tile 对象）
        ├── models.py         # 数据结构定义
        ├── evaluator.py      # 启发式路线评估 + 牌河防守分析 + 最优切牌
        ├── capture.py        # DirectX 截屏（dxcam / mss）
        ├── vision.py         # 牌面图像识别（手牌分割 + 花色颜色判断）
        ├── simulator.py      # 对局模拟器（136 张牌墙、碰/杠/和牌）
        └── mcts.py           # MCTS 搜索（flat 采样 + UCB1 树）
```

## 调用链路

```
DirectX 截屏 (capture.py)
    ↓
图像识别 (vision.py) → 牌面编码
    ↓
编码解析 (parser.py) → Tile 列表
    ↓
决策引擎 ───┬── 启发式评估 (evaluator.py)
            │     ├── 路线评分 + 进张计算
            │     └── 牌河防守分析（安全牌/筋牌/壁牌）
            └── MCTS 搜索 (mcts.py + simulator.py)
    ↓
中文牌名输出 (cli.py) → 最优切牌建议
```

## 4-AI 自对弈模拟

`self_play.py` 提供完整回合制的 4 人自对弈模拟，支持批量统计和单局回放。

```bash
# 50局批量统计
python self_play.py 50

# 单局详细回放（显示每步摸牌、切牌、碰、杠、和牌）
python self_play.py --detail
```

### 4 种差异化 AI 策略

| AI | 策略 | 特点 |
| --- | --- | --- |
| AI-1 | MCTS 搜索 | 每步模拟 200 局，flat 蒙特卡洛选最优 |
| AI-2 | 启发式评估 + 防守 | evaluator 路线评分 + 牌河防守分析 |
| AI-3 | 激进进攻型 | 对子/刻子权重极高，不防守，优先听牌 |
| AI-4 | 保守防守型 | 分析河牌避点炮，听牌后才进攻 |

### 50 局差异化策略统计结果（含防守优化后）

| 策略 | 胜场 | 胜率 | 自摸 | 荣和 | 点炮 | 防守评分 |
| --- | --- | --- | --- | --- | --- | --- |
| **AI-3 (激进进攻型)** | **24** | **48.0%** | 9 | 15 | 4 | -34 |
| AI-1 (MCTS搜索) | 8 | 16.0% | 3 | 5 | 8 | -77 |
| AI-4 (保守防守型) | 5 | 10.0% | 1 | 4 | 9 | -87 |
| AI-2 (启发式+防守) | 3 | 6.0% | 2 | 1 | **4** | **-40** |

> **防守优化效果**：AI-2 点炮从 13 次降至 4 次（-69%），防守评分从 -130 升至 -40。牌河防守分析（安全牌/筋牌/壁牌）已集成到 evaluator 中。

## 人格库系统

`personality_library.py` — 30 种预设打牌人格 + 参数化决策 + 动态切换器。

```bash
# 列出所有 30 种人格
python personality_library.py list

# 200局锦标赛（所有人大乱斗，统计排名）
python personality_library.py tournament 200

# 切换器 vs 静态人格对比
python personality_library.py switcher 100
```

### 人格参数维度

每种人格通过 7 维权重控制决策偏好：

| 维度 | 说明 | 范围 |
| --- | --- | --- |
| 对子权重 | 对子保留价值 | 40-130 |
| 刻子权重 | 刻子保留价值 | 120-260 |
| 顺子权重 | 搭子/顺子加分 | -15~35 |
| 防守权重 | 避点炮倾向 | 0~1.0 |
| 字牌意愿 | 保留字牌倾向 | -40~40 |
| 速攻倾向 | 快速听牌偏好 | 0~1.0 |
| 风险容忍 | 冒险 vs 保守 | 0~1.0 |
